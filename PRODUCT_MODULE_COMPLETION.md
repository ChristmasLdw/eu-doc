## 🔐 权限控制说明

### 角色定义
- **admin（管理员）**：平台管理员，可以操作所有企业的产品
- **user（普通用户）**：企业用户，只能操作自己企业的产品

### 权限矩阵

| 操作 | 管理员 | 企业用户 |
|------|--------|----------|
| 查看所有企业的产品 | ✅ | ❌ |
| 查看自己企业的产品 | ✅ | ✅ |
| 为任意企业创建产品 | ✅ | ❌ |
| 为自己企业创建产品 | ✅ | ✅ |
| 编辑任意企业的产品 | ✅ | ❌ |
| 编辑自己企业的产品 | ✅ | ✅ |
| 删除任意企业的产品 | ✅ | ❌ |
| 删除自己企业的产品 | ✅ | ✅ |

### 权限检查机制

后端通过查询 `company_members` 表验证用户权限：

```javascript
// 检查用户是否是企业成员
const companyMember = db.prepare(
  'SELECT * FROM company_members WHERE user_id = ? AND company_id = ? AND status = "active"'
).get(req.admin.id, company_id);

if (!companyMember) {
  return res.status(403).json({
    success: false,
    message: '无权操作该企业的产品',
  });
}
```

## 🎯 业务流程

### 创建产品流程
1. 用户选择企业
2. 填写产品基本信息（名称、型号、描述等）
3. 可选：选择产品分类
4. 提交创建
5. 后端验证权限（是否有权为该企业创建产品）
6. 创建产品记录
7. 返回产品 ID

### 编辑产品流程
1. 用户打开产品编辑页面
2. 系统加载产品信息
3. 用户修改字段
4. 提交保存
5. 后端验证权限（产品是否属于用户的企业）
6. 更新产品记录
7. 记录审计日志

### 删除产品流程
1. 用户点击删除按钮
2. 确认删除操作
3. 后端验证权限
4. 检查是否有关联文档
5. 执行软删除（status = 'deleted'）
6. 记录审计日志
