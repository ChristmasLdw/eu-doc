## 📝 技术实现细节

### 已实现的模式

#### 1. 菜单翻译
```javascript
// 菜单定义使用 labelKey
const personalMenus = [
  { id: 'profile', labelKey: 'admin.menu.profile' },
  // ...
];

// MenuGroup 组件使用 t() 翻译
function MenuGroup({ items }) {
  const { t } = useTranslation();
  return items.map(item => <span>{t(item.labelKey)}</span>);
}
```

#### 2. 页面内容翻译
```javascript
// Section 标题和描述
<Section title={t('admin.profile.title')} desc={t('admin.profile.desc')}>

// 表单字段
<label>
  <span>{t('admin.profile.name')}</span>
  <input placeholder={t('admin.profile.namePlaceholder')} />
</label>

// 按钮
<button>{t('admin.common.save')}</button>
```

#### 3. 动态数据翻译
```javascript
// 表格表头
<DataTable
  columns={[
    t('admin.security.time'),
    t('admin.security.device'),
    t('admin.security.location')
  ]}
/>

// 状态标签
<span>{t('admin.security.bound')}</span>
```

### 翻译键命名规范
```
admin.{page}.{field}
admin.{page}.{action}
admin.common.{commonAction}
```

示例：
- `admin.profile.name` - 个人资料页面的"姓名"字段
- `admin.profile.saveChanges` - 个人资料页面的"保存修改"按钮
- `admin.common.save` - 通用的"保存"按钮

每次专注完成一个页面的全部翻译，测试后再继续下一个。

1. 在 zh.json 和 en.json 中添加该页面的翻译键

4. 测试构建：`npm run build`
5. 在浏览器中测试语言切换

### 方法2：批量替换（适合重复内容）
对于重复出现的文本（如"保存"、"取消"），可以批量替换：

```bash

```
