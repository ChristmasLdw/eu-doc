## 🎨 设计理念改进

### 问题分析
之前的设计问题：
- ❌ 管理界面使用独立的侧边栏导航，与前台风格不统一
- ❌ 用户在管理界面无法访问前台导航栏功能（搜索、主题切换等）
- ❌ 缺少返回首页的快捷方式

### 新设计方案 ✅
**核心理念：管理界面是前台的扩展，而不是独立系统**

1. **保留顶部导航栏**
   - 所有页面（除登录/注册）都显示导航栏
   - 用户可以随时返回首页、使用搜索、切换主题/语言
   - 导航栏显示登录状态和用户信息

2. **根据角色使用不同布局**
   - **管理员(admin)**: 使用完整的AdminLayout（带侧边栏）- 需要管理证书、企业等
   - **普通用户(user)**: 使用简化的AdminLayoutSimple（无侧边栏）- 只需查看个人上传

3. **统一的视觉风格**
   - 管理页面内容采用卡片式布局
   - 与前台证书详情页风格一致
   - 最大宽度1200px居中显示

## 📝 详细修改

### 1. App.jsx
```javascript
// 之前：所有 /admin 路径都隐藏导航栏
const isAdminPage = location.pathname.startsWith('/admin');
{!isAdminPage && <Navbar />}

// 现在：只在登录/注册页隐藏导航栏
const isLoginOrRegister = location.pathname === '/admin/login' || 
                          location.pathname === '/admin/register';
{!isLoginOrRegister && <Navbar />}

// 根据用户角色选择布局
<AdminRoute>
  {isAdmin ? <AdminLayout /> : <AdminLayoutSimple />}
</AdminRoute>
```

### 2. AdminLayoutSimple.jsx (新增)
```javascript
// 简化布局：去除侧边栏，只保留内容区
export default function AdminLayoutSimple() {
  return (
    <div className={styles.simpleLayout}>
      <main className={styles.simpleContent}>
        <Outlet />
      </main>
    </div>
  );
}
```

### 3. AdminLayout.module.css
新增简化布局样式：
```css
.simpleLayout {
  min-height: calc(100vh - 60px);
  padding-top: 80px; /* 给导航栏留空间 */
}

.simpleContent {
  max-width: 1200px;
  margin: 0 auto;
  padding: 24px;
}
```

### 4. Navbar.jsx
导航栏已有的登录状态逻辑：
- 未登录：显示"注册"和"登录"按钮
- 已登录(管理员)：显示"后台管理"链接 + "退出登录"
- 已登录(普通用户)：显示"我的上传"链接 + "退出登录"

## 🎯 用户体验改进

### 普通用户流程
1. 访问首页 → 看到导航栏

## 💡 设计哲学

这次重构遵循了几个重要的设计原则：

1. **一致性**: 用户在不同页面间切换时，应该感觉是在同一个系统中
2. **可访问性**: 核心功能（搜索、主题切换）应该在任何页面都可访问
3. **渐进增强**: 普通用户看到简化界面，管理员看到完整功能
4. **最小惊讶原则**: 用户期望导航栏始终存在，不应该突然消失
