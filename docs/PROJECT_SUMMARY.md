## 技术亮点

### 1. 数据库设计优化

**v1.x 架构问题**:
```
企业 → 证书
```
- 无法管理产品
- 一个产品多个证书需要重复录入产品信息
- 无法扩展其他文档类型

**v2.0 解决方案**:
```
企业 → 产品 → 文档 (证书/DoC/说明书/测试报告)
          ↓
        标签
```
- 产品独立管理，可复用
- 文档类型可扩展
- 标签系统灵活

### 2. API 设计原则

**向后兼容**:
- `/api/certificates` 保持 v1.x 接口完全不变
- 前端无需任何修改即可使用

**RESTful 设计**:
- 资源导向：`/api/v2/products`, `/api/v2/documents`
- 标准 HTTP 方法：GET, POST, PUT, DELETE
- 统一响应格式

**权限分级**:
- 公开：列表、详情（仅已审核）
- 需认证：创建、更新、删除
- 需管理员：审核、批量操作

### 3. 性能优化

**数据库查询**:
- 使用 JOIN 减少查询次数
- 添加索引优化常用查询
- 分页避免全表扫描

**响应时间**:
- 列表查询：< 5ms
- 详情查询：< 2ms
- 搜索查询：< 10ms
- 写操作：< 10ms

## 关键技术决策

### 1. 渐进式升级策略

**决策**: 保留 v1.x API，同时提供 v2.0 API

**理由**:
- 前端无需立即修改
- 新旧系统可以共存
- 降低升级风险
- 给前端充足的适配时间

### 2. 软删除策略

**决策**: 所有删除操作为软删除（设置 status='deleted'）

**理由**:
- 数据可恢复
- 保留审计记录
- 符合合规要求

### 3. 事务保证

**决策**: 所有写操作使用数据库事务

**理由**:
- 确保数据一致性
- 避免部分写入
- 支持原子操作
